# Shared Code Policy

Shared vision can be centralized early.

Shared code must be earned.

## Rule

```text
Do not extract shared code until at least two projects prove the same need.
```

## Centralize Early

- design principles
- gate templates
- prompts
- knownbugs patterns
- handoff formats
- agent instructions
- CI policies

## Extract Later

- Compose components
- error mappers
- design tokens
- project status components
- Markdown export tools
- shared adapters

## Avoid

- premature shared modules
- app-specific logic hidden in shared code
- unclear ownership
- shared code that forces apps to look or behave the same
