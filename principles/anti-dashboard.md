# Anti-Dashboard

ANVIL projects avoid default dashboard thinking.

A dashboard shows many things because the system can know many things.

ANVIL surfaces should show the relevant truth for the current state.

## Rule

```text
It does not say everything it knows. It says what matters now.
```

## Good Use

- one dominant anchor
- clear next action
- progressive disclosure
- visible state change
- minimal navigation when context is obvious

## Bad Use

- home screens full of cards
- settings-first workflows
- tab-heavy layout before the task exists
- showing stats instead of helping the user act
- making the user scan instead of understand
