# Template Change Gate

Dieser Mini-Prozess gilt für Änderungen an Template-Artefakten.

1. **Vorschlag**
   - Quelle dokumentieren
   - Nutzen beschreiben
   - Zielpfad benennen

2. **Ballast-Check**
   - Gegen `sources/template-extraction/template-exclusion-rules.md` prüfen

3. **Scope-Entscheid**
   - Entscheidung treffen: **Core** vs. **Modul**

4. **Pilot**
   - Änderung in **genau einem** Template umsetzen

5. **Review**
   - Nach **1–2 realen Nutzungen** bewerten

6. **Promotion**
   - Erst danach Promotion in **Core**
