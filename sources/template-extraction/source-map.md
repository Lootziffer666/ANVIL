# Template Extraction Source Map

## PAINKILLER

Useful for:

- gated implementation discipline
- CI-first Android verification
- `AGENTS.md` / `claude.md` split
- `knownbugs.md` append-only risk memory
- handoff files per gate
- PR / issue templates
- explicit hard non-goals
- safety-first GitHub write orchestration
- environment-only blocker handling

## DEAFPIPER

Useful for:

- end-to-end product pipeline from idea to launch / iteration
- stage map with inputs, outputs, quality gates, and templates
- `work/` artifact folders per pipeline stage
- CustomGPT / agent configuration registry pattern
- one-output-per-stage reliability rule
- cross-cutting modules such as translation, slides, and brand guardian

## OPENDORK

Useful for:

- runtime profile concept (`interactive`, `batch`, `overnight_safe`, `overnight_aggressive`)
- provider/model catalog
- failover / cooldown / cache / budget guard concepts
- validation profiles
- artifact classes: raw / validated / rejected / gold / diffs / reports / replays
- spend and attempt logs

## CATALON-GUARD

Useful for:

- reminder that a short README is not enough for agent-safe development
- test-first / guard-framework naming lineage
- cautionary source: templates must not overclaim from vague repo docs
- reinforces need for explicit instructions, build commands, and current-status files
